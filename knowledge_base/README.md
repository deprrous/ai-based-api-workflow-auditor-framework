# Knowledge Base

This directory holds the reference material the system uses to reason about API risk.

## Planned internal areas

- `owasp_api_top_10/` - mapped concepts and reference material.
- `detection_rules/` - rule definitions and heuristics.
- `prompt_assets/` - prompt snippets and reusable knowledge fragments.

The knowledge base should remain auditable and easy to extend for open-source contributors.
